# Blueprint for the Manual Editor
This is a guide made using the Quarto documentation suite, which is an open-source scientific and technical publishing system. The purpose of this guide is to provide a standardized guideline for the look and feel of the manuals edited using the Manual Editor tool.

# 📘 Quarto Project

## What is Quarto?

**Quarto** is an open-source publishing system designed for creating dynamic documents, reports, presentations, websites, and blogs. It supports multiple programming languages including R, Python, Julia, and Observable, and integrates seamlessly with Jupyter and R Markdown.

With Quarto, you can:

- 🧠 Combine code, output, and narrative in a single document
- 📊 Create reproducible reports and dashboards
- 🎨 Customize styling with themes and templates
- 🌐 Publish to HTML, PDF, Word, or full websites

---

## 🚀 Getting Started

To get started with Quarto:

1. **Install Quarto**  
   Download and install from [quarto.org](https://quarto.org/docs/get-started/)

2. **Clone this repository**  
   ```bash
   git clone https://github.com/jugazabal/manual.git
   cd manual
   ```

3. **Render the document**  
   Run the following command to render your `.qmd` file:
   ```bash
   quarto render
   ```
